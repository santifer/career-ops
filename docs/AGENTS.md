# Docs Context

`docs/` is system-layer documentation.

Use docs for architecture, setup, scripts, customization guidance, and implementation notes that are safe to share across users.

Do not put this user's private CV details, reports, application status, compensation preferences, or personal strategy into docs. Those belong in user-layer files such as `config/profile.yml`, `modes/_profile.md`, `reports/`, or `interview-prep/`.

When documenting a workflow, prefer stable commands and real file paths relative to the repository root.
