# Docs Navigation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a contributor-friendly documentation map that explains every markdown file in the repository, how the docs fit together, and where user-specific customization belongs.

**Architecture:** Keep the existing repository structure intact and add a documentation navigation layer on top of it. Create a docs hub, create an exhaustive markdown file map, add a modes index, and patch the existing README/customization docs so they route contributors into the new map and align with the data contract.

**Tech Stack:** Markdown, existing repository docs, shell verification commands
