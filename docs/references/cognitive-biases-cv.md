# Cognitive Biases in CV Processing

How cognitive shortcuts affect recruiter decisions, and how to design for them.

## The 6-Second Gate

The CV is not a document about you. It is a visual stimulus designed to activate a network of positive associations in the recruiter's brain within 6 seconds. Each element is an independent variable affecting the dependent variable: "interview invitation."

CVs don't predict job performance (Schmidt & Hunter, 1998). They predict **who gets the interview**. This is pattern-matching, not depth-evaluation.

## F-Pattern Scanning (Neilsen, 2006)

Eye-tracking studies show recruiters follow an F-pattern:
1. Horizontal scan across the top (name, title, summary)
2. Vertical scan down the left margin (section headers, role titles)
3. Partial horizontal scans at section starts (first bullets)

**Implication:** The top-left quadrant of your CV is prime real estate. Every word there must earn its place.

## Cognitive Load (Sweller, 1988; Miller, 1956)

Recruiters process hundreds of CVs per day. Capacity is saturated. Each bullet requiring re-reading, each unconventional format, each ambiguous phrase increases extraneous load and decreases retention probability.

Miller's Law: working memory holds 7±2 chunks. A 2-page CV with 30+ unstructured bullets exceeds this limit and collapses.

**Implication:**
- First page: 8-12 bullets maximum
- Zero bullets requiring re-reading
- Header: ≤5 information chunks
- Useful content ratio ≥ 70% (cut noise)

## Key Biases

### Halo Effect (Thorndike, 1920)
One strong positive impression ("worked at Nubank") colors evaluation of all other dimensions. The first impressions list (Section 1a) is the halo gate — if the top is strong, the recruiter interprets the rest as strong.

**Design for it:** Place your strongest signal in the first 3 lines. That signal anchors everything that follows.

### Anchoring (Tversky & Kahneman, 1974)
The first element read anchors all subsequent interpretation. If the top bullet shows impact, the reader is primed to see impact throughout.

**Design for it:** Your #1 bullet under each role must be your E5 claim. Never lead with a task description.

### Confirmation bias (Nickerson, 1998)
Recruiters seek to confirm their first impression. If the scan is positive, they read for strengths. If negative, they read for red flags.

**Design for it:** No red flags in the first 6 seconds means no typos, no unexplained gaps at the top, no generic adjectives.

### Loss Aversion (Kahneman & Tversky, 1979)
Red flags weigh 2x more than green flags. A single typo can eliminate. A missing metric can downgrade.

**Design for it:** Eliminate all red flags before adding new green flags. Fix what's wrong before adding what's impressive.
