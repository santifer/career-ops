## 4. Patterns learned

### Offshore staffing trap
Hire Hangar, Hire with Near, Trektel = offshore staffing intermediaries que pagan a talento LATAM ~$800–2,000/mes vendiendo "30-70% savings" a clientes US. **Auto-flag cualquier cosa bajo $3,500 floor.** Empresas reales con stack nombrado + comp creíble puntúan mucho más alto.

### Web hosting ≠ candidate's domain
Hostinger (web hosting: DNS, cPanel, hPanel, email) es un dominio fuera del perfil de Nestor (SaaS/APIs/payments/automation). El tag "LATAM" atrae pero el dominio no transfiere. Bajar score por gap estructural de dominio.

### Freelance/contractor comp check
Varias ofertas LATAM "remote" son contractor (sin beneficios, sin seguro, sin pensión). Restar ~20% al comparar contractor vs empleo formal. Confirmar tipo de contrato + rango antes de invertir tiempo (Hostinger, Mercor, posiblemente Alegra).

### LinkedIn recruiter outreach (premium accounts)
Muchas reclutadoras tienen LinkedIn Premium → no se les puede enviar mensaje directo gratis. PERO sí se puede adjuntar **nota en la solicitud de conexión, máximo ~200 caracteres.** Redactar mensajes cortos: rol exacto + pregunta crítica (ubicación/contractor) + 1 hook de fit. Salario se pregunta en el chat tras conectar, no en la nota.

### AI-first roles favor Nestor's agent-building background
Roles que piden "uso activo de AI / construir con AI" (Alegra, Mercor, Hostinger CX AI Engineer path) son fit fuerte por los agentes en producción de dLocal. Encuadrar como "construyo con AI, no solo la uso".

## 6. Technical setup & workarounds (DON'T re-discover these)

### PDF generation — use the wrapper, NOT generate-pdf.mjs directly
`generate-pdf.mjs` tiene bug de Windows `isMain`: `import.meta.url` (`file:///C:/...`) nunca iguala `resolve(process.argv[1])` (`C:\...`), así que correrlo directo **sale en silencio sin PDF**. Usar el wrapper:
```bash
node run-pdf.mjs "C:/tmp/cv-nestor-velasquez-{company}.html" "C:/Proyectos_IA/career-ops/output/cv-nestor-velasquez-{company}-{date}.pdf" "--format=letter"
```
- Formato **letter** (no a4). Target **2 páginas**.
- Playwright Chromium ya instalado.
- HTML del CV tailored vive en `C:\tmp\cv-nestor-velasquez-{company}.html`; PDFs en `output/`.
- Template usa **rutas absolutas de fuentes**: `C:/Proyectos_IA/career-ops/fonts/space-grotesk-latin.woff2` y `dm-sans-latin.woff2`.
- Para caber en 2 páginas: body 10.5px, bullets 10px, line-height ~1.45, padding 0.5in top, Education+Certifications merged, máx 2 proyectos.

### markitdown (leer docs binarios sin quemar tokens)
- Instalado vía `pip install 'markitdown[all]'`. Skill global en `~/.claude/skills/markitdown.md`.
- Convertir PDFs/DOCX/PPTX/imágenes con `markitdown "path"` antes de leer. Helper: `node read-doc.mjs CV/file.pdf` (o una carpeta entera).

### Scan workflow
- `node scan.mjs` = Level 0+2 zero-token (parsers locales + APIs Greenhouse/Ashby/Lever). Corre primero.
- Level 3 (WebSearch) = lanzar subagente `general-purpose` en background para no quemar contexto del agente principal. Cubre empresas sin API (Stripe, HubSpot, Datadog, Cloudflare, etc.).
- Reportes de evaluación: `node reserve-report-num.mjs` (claim número), escribir reporte, `node reserve-report-num.mjs --release {n}`.

### Shell notes (Windows)
- Usar **Bash tool** para git/file ops y rutas `/c/...`; **PowerShell** para Node con rutas `C:\...`.
- Comillas en rutas con espacios.
- WebFetch a Lever/LinkedIn/Glassdoor suele dar 403/999 → usar WebSearch + agregadores (startup.jobs, himalayas.app, talent.com) para extraer JD/comp.
