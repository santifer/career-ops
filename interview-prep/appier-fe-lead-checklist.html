React.memo / useCallback / useMemo 差別
`React.memo`：包整個 component，props 沒變就跳過 re-render。`useCallback`：穩定 function reference 避免下游 memo 失效。`useMemo`：cache 計算結果，避免昂貴運算每次跑。三者都靠 dep array — 用錯反而是 bug 來源。
useEffect cleanup function 何時跑
每次 effect 重新執行前都跑一次（不是只在 unmount）。所以 dep 變動會：先跑 cleanup → 再跑 new effect。unmount 時跑最後一次 cleanup。常用於：取消事件監聽、cancel fetch、clear timer。

「沒大量 production 跑過——但理解 GraphQL 的核心：client 自己選 fields 解決 REST over-fetching、單一 endpoint 配 schema introspection、N+1 由 resolver + DataLoader 解。Apollo Client 跟 urql 我會選 urql for bundle size。要上手不會花太久。」

CORS preflight 為什麼要 OPTIONS
瀏覽器在「非簡單請求」（custom header / 非 GET-POST、Content-Type 非 form/text）前先發 OPTIONS 探，問 server「我這個 origin / method / header 你接不接？」server 回 `Access-Control-Allow-*`，瀏覽器才放行真正請求。是瀏覽器的安全限制，不是 server 的。
Cookie / localStorage / sessionStorage
Cookie：每 request 自動帶、4KB 上限、有 HttpOnly + SameSite + Secure 等安全 flag、適合 session。localStorage：5–10MB、不會帶 server、persist 直到手動清、適合 user setting。sessionStorage：tab 關就消、不跨 tab、適合臨時表單草稿。
CSS：水平垂直置中三種寫法
① flex：`display:flex; justify-content:center; align-items:center`。② grid：`display:grid; place-items:center`。③ position：`position:absolute; top:50%; left:50%; transform:translate(-50%,-50%)`。現代 prefer grid `place-items:center` — 最短。
Tailwind vs CSS-in-JS trade-off
Tailwind：build-time、零 runtime cost、bundle size 可預測（PurgeCSS）、design system 一致性靠 config。CSS-in-JS（styled-components / emotion）：runtime overhead、dynamic style 容易、SSR 要處理 critical CSS。Appier 用 Tailwind 我直接上，剛好對位。
