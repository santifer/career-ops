## 新創經歷量化指南

他在 Key Highlights 已有 "up to 80%" 和 "within 3 months" 兩個數字，但未錨定回 Experience bullet。修正方向是兩步驟：① 把現有數字帶回對應的 bullet；② 補充其他可取得的量化項。以下是可取得的數字，從最易到最難：

**交付速度（最容易）**
- 幾個月從 0 上線 → 他有 3 個月，但只放在 Key Highlights，未出現在 Experience bullet
- 發布頻率變化：e.g. 從每月 1 次 → 每週 2 次

**產品規模（容易）**
- 同時運行幾個白牌產品 / 幾個獨立部署環境
- 支援幾個地區 / 市場
- MAU / DAU（即使是區間，"10 萬+" 夠用）

**團隊效率（中等）**
- On-demand environment 讓 QA 等待時間從 X 縮短到 Y
- PR merge time 或 code review cycle 改善幅度
- CI pipeline 執行時間縮短多少

**系統可靠性（中等）**
- 監控平台將異常偵測時間從幾小時 → 幾分鐘
- 部署失敗率降低多少
- Incident response time 改善幅度

**技術指標（視情況）**
- Bundle / build size 縮小多少
- 內部 component library 取代幾個第三方套件

**天譽現有 bullet 最快可補的數字：**

| 現有 bullet | 可追問的 outcome |
|------------|----------------|
| Micro-frontend 架構 | 支援幾個獨立產品？上線幾個白牌客戶？ |
| CI/CD + on-demand env | 部署頻率提升多少？測試等待時間縮短？ |
| 域名監控平台 | 異常偵測時間從 X 到 Y？覆蓋幾個 region？ |
| 0→1 交付 | 幾個月？幾個工程師？幾個市場同時上線？ |
