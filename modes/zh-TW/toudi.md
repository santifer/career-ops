# 模式：toudi -- 投遞即時助手

候選人在瀏覽器填寫投遞表單時的互動模式。讀取畫面上顯示的內容，載入先前評估的脈絡，為表單的每個問題生成個人化回答。

## 需求

- **有 Playwright 最佳**：候選人可以看到瀏覽器，Claude 可以與頁面互動。
- **沒有 Playwright**：候選人分享截圖或手動貼上問題。

## 工作流程

```
1. 偵測    → 讀取 Chrome 活躍分頁（截圖/URL/標題）
2. 辨識    → 從頁面擷取公司名 + 職位
3. 搜尋    → 在 reports/ 中匹配既有報告
4. 載入    → 讀取完整報告 + 區塊 G（若存在）
5. 比對    → 畫面上的職缺與已評估的是否一致？有變更 → 通知
6. 分析    → 辨識表單中顯示的所有問題
7. 生成    → 為每個問題生成個人化回答
8. 呈現    → 以方便複製貼上的格式顯示回答
```

## 步驟 1 -- 偵測職缺

**有 Playwright：** 擷取活躍頁面的 snapshot。讀取標題、URL、顯示內容。

**沒有 Playwright：** 請候選人：
- 分享表單截圖（Read tool 可以讀取圖片）
- 或將表單問題以文字貼上
- 或告知公司名 + 職位以便搜尋脈絡

## 步驟 2 -- 辨識脈絡並搜尋

1. 從頁面擷取公司名和職位名稱
2. 在 `reports/` 中搜尋公司名（Grep，不分大小寫）
3. 有匹配 → 載入完整報告
4. 有區塊 G → 載入先前的回答草稿作為基礎
5. 沒有匹配 → 通知並建議執行快速 auto-pipeline

## 步驟 3 -- 偵測職缺變更

畫面上的職缺與已評估的不同時：
- **通知候選人：**「職位已從 [X] 變更為 [Y]。要重新評估還是將回答調整到新職稱？」
- **調整的話：** 不重新評估，將回答調整到新職稱
- **重新評估的話：** 執行完整 A-F 評估、更新報告、重新生成區塊 G
- **更新 tracker：** 必要時變更 `applications.md` 中的職位名稱

## 步驟 4 -- 分析表單問題

辨識所有顯示的問題：
- 自由文字欄（自我介紹、為什麼想加入、動機等）
- 下拉選單（如何得知此職缺、工作資格等）
- 是/否（是否願意搬遷、簽證、可到職日等）
- 薪資欄（期望薪資、薪資範圍）
- 上傳欄位（履歷、求職信 PDF）

分類每個問題：
- **區塊 G 已回答** → 調整既有回答
- **新問題** → 從報告 + `cv.md` 生成回答

## 步驟 5 -- 生成回答

為每個問題生成回答，遵循：

1. **報告脈絡**：使用區塊 B 的 proof point、區塊 F 的 STAR 故事
2. **先前的區塊 G**：有草稿就以此為基礎精煉
3. **「我在選你們」的語調**：與 auto-pipeline 相同的框架 -- 自信但不卑微
4. **具體性**：引用畫面上可見的 JD 具體內容
5. **career-ops proof point**：如果有「其他資訊」欄位就放入

**台灣市場常見的表單特定問題：**
- **期望薪資（月薪/年薪，含稅前）** → 使用 `profile.yml` 的範圍，以新台幣表示，附註「視整體 package 可議」
- **僱用類型偏好（正職/約聘/承攬）** → 依 `profile.yml` 回答，或適用時回答「皆可配合」
- **最快可到職日** → 考量目前的預告期後的實際日期（正職依年資：10-30 天）
- **工作資格** → 清楚回答。台灣國籍：「中華民國國籍，無需工作許可」
- **語言能力** → 逐語言列出程度（母語、精通、中級、初級）
- **可否到辦公室** → 依 `profile.yml` 的地點設定回答

**輸出格式：**

```
## [公司名] -- [職位名稱] 的回答

依據：Report #NNN | 分數：X.X/5 | 人才定位：[類型]

---

### 1. [表單問題（原文）]
> [可直接複製貼上的回答]

### 2. [下一個問題]
> [回答]

...

---

備注：
- [關於職缺的觀察、變更等]
- [候選人應檢視的個人化建議]
```

## 步驟 6 -- 投遞後（選用）

候選人確認已送出投遞時：
1. 將 `applications.md` 的狀態從「Evaluated」更新為「Applied」
2. 用最終回答更新報告的區塊 G
3. 建議下一步：使用 `/career-ops contacto` 進行 LinkedIn 聯繫

## 捲動處理

表單問題超出可見範圍時：
- 請候選人向下捲動並分享另一張截圖
- 或貼上其餘問題
- 反覆處理直到涵蓋整個表單
