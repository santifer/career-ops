# Режим: pdf — Генерация ATS-оптимизированного PDF

## Полный пайплайн

1. Прочитать `cv.md` как источник правды
2. Запросить JD у пользователя, если его нет в контексте (текст или URL)
3. Извлечь 15-20 ключевых слов из JD
4. Определить язык JD → язык CV (RU по умолчанию для русских JD)
5. Определить локацию компании → формат бумаги:
   - US/Canada → `letter`
   - Россия и остальной мир → `a4`
6. Определить архетип роли → адаптировать фрейминг
7. Переписать Professional Summary, внедрив ключевые слова из JD
8. Выбрать топ 3-4 наиболее релевантных проекта
9. Перестроить буллеты опыта по релевантности к JD
10. Построить competency grid из требований JD (6-8 keyword phrases)
11. Внедрить ключевые слова естественно в существующие достижения (НИКОГДА не выдумывать)
12. Сгенерировать полный HTML из шаблона + персонализированный контент
13. Записать HTML в `/tmp/cv-candidate-{company}.html`
14. Выполнить: `node generate-pdf.mjs /tmp/cv-candidate-{company}.html output/cv-candidate-{company}-{YYYY-MM-DD}.pdf --format={letter|a4}`
15. Отчитаться: путь к PDF, кол-во страниц, % покрытия ключевых слов

## Выбор шаблона

- **Для англоязычных JD**: использовать `templates/cv-template.html`
- **Для русскоязычных JD**: использовать `templates/cv-template-ru.html` (если существует), иначе `templates/cv-template.html` с русскими секциями

## Правила ATS (чистый парсинг)

- Layout single-column (без сайдбаров, без параллельных колонок)
- Стандартные заголовки: "О себе" / "Professional Summary", "Опыт работы" / "Work Experience", "Образование" / "Education", "Навыки" / "Skills", "Сертификации" / "Certifications", "Проекты" / "Projects"
- Без текста в изображениях/SVG
- Без критической информации в headers/footers PDF (ATS их игнорирует)
- UTF-8, текст выделяемый (не растеризованный)
- Без вложенных таблиц
- Ключевые слова из JD распределены: Summary (топ 5), первый буллет каждой роли, Skills

## Дизайн PDF

- **Шрифты**: Space Grotesk (заголовки, 600-700) + DM Sans (текст, 400-500)
- **Шрифты self-hosted**: `fonts/`
- **Header**: имя в Space Grotesk 24px bold + линия градиент `linear-gradient(to right, hsl(187,74%,32%), hsl(270,70%,45%))` 2px + строка контактов
- **Заголовки секций**: Space Grotesk 13px, uppercase, letter-spacing 0.05em, цвет cyan primary
- **Body**: DM Sans 11px, line-height 1.5
- **Названия компаний**: accent purple `hsl(270,70%,45%)`
- **Поля**: 0.6in
- **Фон**: чисто белый

## Порядок секций (оптимизирован под "6-секундный рекрутерский скан")

1. Header (крупное имя, градиент, контакты, ссылка на портфолио)
2. О себе / Professional Summary (3-4 строки, насыщены ключевыми словами)
3. Ключевые компетенции / Core Competencies (6-8 keyword phrases в flex-grid)
4. Опыт работы / Work Experience (обратная хронология)
5. Проекты / Projects (топ 3-4 наиболее релевантных)
6. Образование и сертификации / Education & Certifications
7. Навыки / Skills (языки + технические)

**Для русскоязычных CV дополнительно включить** (если указано в profile.yml):
- Город проживания
- Гражданство
- Готовность к переезду / командировкам

## Стратегия keyword injection (этично, основано на правде)

Примеры легитимной переформулировки:
- JD говорит "RAG pipelines", а CV говорит "LLM workflows with retrieval" → изменить на "проектирование RAG pipeline и оркестрация LLM workflows"
- JD говорит "highload", а CV говорит "обработка большого объёма данных" → изменить на "highload-системы: обработка 50М+ запросов/день"
- JD говорит "микросервисы", а CV говорит "распределённые системы" → изменить на "микросервисная архитектура, распределённые системы"

**НИКОГДА не добавлять навыки, которых у кандидата нет. Только переформулировать реальный опыт словами JD.**

## Canva CV Generation (опционально)

Если в `config/profile.yml` есть `canva_resume_design_id`, предложить выбор:
- **"HTML/PDF (быстро, ATS-оптимизировано)"** — основной поток выше
- **"Canva CV (визуальный, сохраняет дизайн)"** — поток через Canva

Если нет `canva_resume_design_id` — пропустить и использовать HTML/PDF.
