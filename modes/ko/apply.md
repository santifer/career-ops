# 모드: apply -- 라이브 지원서 도우미

후보자가 브라우저에서 지원서를 작성하는 동안 화면을 읽고, 기존 평가 컨텍스트를 불러와 질문별로 맞춤 답변을 생성하는 인터랙티브 모드입니다.

## Requirements

- **Playwright 가시 모드 권장:** 브라우저 상태를 직접 읽고 상호작용할 수 있습니다.
- **Playwright 없이도 가능:** 스크린샷 공유 또는 질문 텍스트 붙여넣기 방식으로 진행합니다.

## Workflow

```
1. DETECT     → 현재 탭의 제목 / URL / 화면 내용 읽기
2. IDENTIFY   → 회사명 + 직무명 추출
3. LOOK UP    → reports/ 에서 기존 평가 찾기
4. LOAD       → report 전체 + Section G 초안 불러오기
5. COMPARE    → 현재 직무와 기존 평가가 같은지 확인
6. ANALYZE    → 보이는 질문 전부 식별
7. GENERATE   → 질문별 맞춤 답변 생성
8. PRESENT    → 바로 복붙 가능한 형태로 출력
```

## Step 1 -- 공고 감지

**Playwright 사용 시:** 활성 페이지 snapshot을 읽고 title, URL, 본문을 확인합니다.

**Playwright가 없으면:** 후보자에게 아래 중 하나를 요청합니다.

- 지원서 스크린샷 공유
- 질문 텍스트 붙여넣기
- 회사명 + 직무명 제공

## Step 2 -- 컨텍스트 로드

1. 화면에서 회사명과 직무명을 추출
2. `reports/`에서 회사명 기준으로 검색
3. 매칭되면 해당 report 전체를 로드
4. Section G가 있으면 초안 답변도 함께 읽기
5. 매칭이 없으면 경고하고 빠른 auto-pipeline 실행을 제안

## Step 3 -- 직무 변경 감지

현재 화면의 직무가 기존 평가와 다르면:

- 후보자에게 경고: "직무가 [X]에서 [Y]로 바뀌었습니다. 다시 평가할까요, 아니면 기존 답변을 새 직무 기준으로 조정할까요?"
- **조정**: 재평가 없이 답변만 새 제목에 맞춰 수정
- **재평가**: A-F 전체 재실행, report 갱신, Section G 재생성
- 필요하면 tracker의 직무명도 업데이트

## Step 4 -- 질문 분석

보이는 질문을 전부 식별합니다.

- 자유서술형(지원 동기, 자기소개, 왜 이 회사인지 등)
- 드롭다운(지원 경로, 취업 가능 여부 등)
- 예/아니오 질문(이전 근무 경험, relocation, visa 등)
- 연봉 관련 필드
- 업로드 필드(이력서, 포트폴리오, 커버레터 PDF)

각 질문을 분류:

- **Section G에 이미 답이 있음** → 기존 답변을 현재 문맥에 맞게 다듬기
- **새 질문** → report + `cv.md` 기반으로 새로 생성

## Step 5 -- 답변 생성

각 질문에 대해 아래 순서로 답변을 만듭니다.

1. **report 컨텍스트:** 블록 B의 proof point, 블록 F의 STAR 스토리 활용
2. **Section G 초안:** 이미 있으면 기반으로 사용하되 더 구체화
3. **"제가 이 회사를 고른다" 톤:** 자신감 있고 상호 선택적인 톤
4. **구체성:** 화면에 보이는 JD 요소를 한 가지 이상 반영
5. **career-ops proof point:** 관련 필드가 있으면 강한 근거 한 줄 추가

**한국 지원서에서 자주 나오는 필드:**

- **희망 연봉** → `profile.yml` 범위를 기준으로 KRW 제시, 기본급/총보상 구분
- **고용 형태 선호** → 정규직/계약직/프리랜서 선호를 사실대로
- **입사 가능일** → 실제 notice period를 반영한 날짜 또는 기간
- **취업 가능 여부 / 비자** → 한국 취업 가능 상태를 명확히 설명
- **병역 사항** → 관련될 때만 사실 그대로, 과도한 설명 없이
- **언어 역량** → 한국어/영어 등 업무 수준 기준으로 정리

**출력 형식:**

```markdown
## [회사명] -- [직무] 지원서 답변

Base: Report #NNN | Score: X.X/5 | Archetype: [type]

---

### 1. [폼에 적힌 정확한 질문]
> [바로 붙여넣을 수 있는 답변]

### 2. [다음 질문]
> [답변]

...

---

Notes:
- [직무 변경, 확인 필요 사항 등]
- [후보자가 최종 제출 전에 검토할 포인트]
```

## Step 6 -- 지원 후(선택)

후보자가 실제 제출을 완료했다고 확인해 주면:

1. `applications.md` 상태를 `Evaluated`에서 `Applied`로 업데이트
2. report의 Section G에 최종 답변 반영
3. 다음 단계로 `/career-ops outreach` 제안

## Scroll handling

질문이 화면 아래 더 있으면:

- 추가 스크린샷을 요청하거나
- 남은 질문을 붙여넣게 하고
- 전부 커버될 때까지 반복 처리합니다
