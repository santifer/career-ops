# Canonical Summary Template

Use this structure for `docs/exec-plans/summaries/summary-<workstream>.md`:

```md
# <Workstream Summary>

**Status:** active | completed
**Scope Covered:** <time range or plan set>

## Background

## Scope Covered

## Key Decisions

## Implemented Changes

## Verification Completed

## Open Issues

## Next Recommended Steps

## Archived References
- `docs/exec-plans/archive/<file>.md`
```

Keep the summary short enough to restore context quickly. Preserve only durable decisions, verification, and unresolved risks.
