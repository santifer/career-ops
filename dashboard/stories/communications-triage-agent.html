The architecture I settled on was deliberately constrained: three prompts, one agent, conditional knowledge base loading. Conditional KB loading meant the agent wasn't pulling the full knowledge base on every request; it loaded only the relevant context slice for the tier it had classified, which cut latency and kept the outputs clean.
