Fill operational gaps + lightweight process before permanent systems

Source: Anthropic — Strategic Operations Manager, Claude Marketplace Humanize: 7% unknown

The operational gap appeared before anyone had time to name it. The Office of Cross-Google Engineering supports roughly 1,000 senior technical ICs at the Principal, Distinguished, and Fellow tiers — the top 0.5% of a 180,000-person engineering organization. Inbound communications requests were landing without a triage layer, consuming senior staff time on traffic that ranged from genuinely high-stakes executive escalations to requests that should have been resolved in under three minutes. There was no system sorting one from the other. There was just volume, and people absorbing it manually.

The brief I gave myself was narrow and specific: build something that classifies, resolves, or escalates that inbound stream without human intervention — and does it without sacrificing voice fidelity on the responses it handles autonomously. The second constraint was the harder one. The population this system served had worked with senior communications staff long enough to have calibrated expectations. An agent that classified accurately but wrote like a template would fail the moment it hit send. So the architecture had to hold two things simultaneously: routing logic precise enough to be trusted without supervision, and output quality disciplined enough to represent the office.

The design I settled on was a three-prompt architecture — triage, revise, escalate — with conditional knowledge-base loading at each stage. Inbound traffic lands in triage first, where the agent categorizes it into Low, Medium, or High touch. Low-touch requests get resolved autonomously using Smart Brevity principles, with documented rationale for why the traffic was classified at that tier — an audit layer, not decoration. Medium-touch requests get the revise prompt, which drafts a response against the relevant KB and flags judgment calls for human review. High-touch escalations route to a human immediately, with a brief from the agent summarizing what it already knows about the thread. The KB loading is conditional — it doesn't pull the full corpus for every inbound request, it loads what's relevant to the classification tier and the content category. That distinction matters for latency and for output coherence. A system that over-retrieves writes like it over-retrieved.

I built this on a single-agent Gemini design, not multi-agent orchestration. The three-prompt sequence handles what a multi-agent architecture would overcomplicate. One of the persistent mistakes in early agentic deployments is reaching for complexity before exhausting what a well-scoped single agent can do. The triage system didn't need a constellation of specialized agents. It needed a routing architecture with enough classification depth to be trusted at production and enough output discipline to represent senior communications quality without a human in the loop on resolved traffic.

The result was about 160 operational hours recaptured per year, at greater than 90% classification accuracy across the full inbound stream. That number is a production measurement, not a benchmark run in a controlled environment — it comes from sustained operational deployment against real traffic. The more consequential test came later, in early 2026, when a multi-month medical leave removed the primary human from the system entirely. The agent maintained 100% operational continuity across that absence. Not degraded continuity. Not continuity with occasional intervention. No single escalation that required reaching back to the person who built it. The system ran unattended because the architecture was designed to run unattended — the leave didn't reveal a gap, it proved the design.

What I take from this has less to do with the agent itself and more to do with the discipline that built it. The newsroom decade that preceded this work — live broadcasts running on broken rundowns, editorial operations under active litigation scrutiny, field production executed from a backpack during active civil confrontations — trained a specific instinct: design for the moment when you're not there. The best operational system is the one that doesn't need you to function. The triage agent is the cleanest expression of that instinct I've built in a technology context. The permanent system it was meant to precede never arrived. The lightweight process became the infrastructure.
