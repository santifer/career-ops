Establish infrastructure that keeps comms running at global scale

Source: ElevenLabs — Communications Manager Humanize: 7% low

The comms infrastructure for the Office of Cross-Google Engineering had no floor. Requests came in from roughly 1,000 Principal, Distinguished, and Fellow engineers — the top 0.5% of a 180,000-person organization — and landed in a queue with no classification logic, no routing rules, and no way to distinguish the urgent from the administrative. Every inbound item required a human decision. That was the baseline I inherited.

The first thing I built wasn't an agent. It was a governance layer — because you can't automate a system that doesn't know what it is. The xGE Approvals Matrix defined what kinds of requests warranted executive review, what could be resolved at the communications layer, and what the criteria for escalation actually were. Before that document existed, "needs VP eyes" was a judgment call made fresh every time. After it, that call had a decision tree. I paired the matrix with a 7-day Comment Freeze window and a standing rejection protocol for feedback that lacked data-driven rationale — meaning subjective input that couldn't be tied to a stated objective was returned, not incorporated. That last piece was the most friction-generating and the most necessary. The senior IC population is deeply opinionated. Without a feedback standard that required specificity, every draft became a negotiation rather than a decision.

Once the governance layer was stable — once the rules of the system were documented and defensible — the agent had something to route against. The triage agent I built and deployed in xGE runs a three-prompt architecture: triage, revise, escalate. Each inbound request hits the triage prompt first, which classifies it into Low, Medium, or High touch tiers using conditional knowledge-base loading tuned to the request type. Low-touch items resolve without human review, drafted to Smart Brevity standards with documented rationale for every compression decision. Medium-touch items get a revised draft staged for light human review. High-touch items escalate with a brief on why — the context a human needs to step in fast, not reconstruct from scratch. The agent doesn't guess; it routes against the criteria the Approvals Matrix established. The governance layer is the agent's operating logic, made executable.

What that combination produced: a 50% reduction in low-value, ad hoc executive requests — the noise cleared by the governance layer before it ever reached the agent — and approximately 160 operational hours per year recaptured at above 90% classification accuracy across the full inbound stream. Those aren't prototype numbers. That agent is in production. And the harder validation came during a multi-month medical leave of absence in early 2026: the system maintained 100% operational continuity without a single human intervention across the full absence window. That's the real stress test — not whether an agent works when you're watching it, but whether it holds when you're not there at all.

The thing I'd name in retrospect is that the infrastructure and the agent were the same project, not two separate ones. The governance layer wasn't prep work for the automation. It was the automation — made legible enough that a language model could execute it. Every communications system I've built over the past fifteen years has worked this way: the durable part isn't the tool, it's the decision logic underneath it. The tool just makes that logic run without you in the loop.
